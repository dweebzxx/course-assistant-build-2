Module 05: Overview

This module introduces the normalization principle for minimizing data redundancy and ensuring data consistency. The normalization principles (or norms) helps you understand why, and how, to structure your data for different applications. This module also introduces practical relational database design using MySQL workbench.  

Learning Objectives

  • Explain the purpose and benefits of database normalization
  • Identify data anomalies caused by poor database design
  • Understand and apply normal forms to structure relational data
  • Analyze functional dependencies to guide normalization
  • Evaluate when to normalize vs. denormalize
  • Use MySQL Workbench to create a database
